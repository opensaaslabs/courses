# Runtime Contract

The runtime contract defines how external systems may consume the academy graph.

## Consumers

- Websites
- APIs
- Search engines
- Analytics systems
- Credential issuers
- AI agents
- LMS platforms

## Guarantees

- Stable IDs
- Explicit relations
- Generated exports
- Schema-valid source artifacts
- Governed lifecycle state

## Non-Goals

The academy core does not require a specific runtime, web framework, database, or programming language.
