# AEP-0004: Architecture Governance

## Status
Accepted

## Summary
Introduce architecture governance for principles, domains, capabilities, glossary, taxonomy, quality attributes, reference models, and decision framework.

## Motivation
A CNCF-style specification project needs a stable vocabulary and architecture model so future schemas, contracts, implementations, credentials, publishing surfaces, and agents remain coherent.

## Proposal
Create an `architecture/` domain containing principles, capability model, domain model, glossary, taxonomy, reference models, quality attributes, and decision framework.

## Compatibility
No breaking changes.

## Security and Privacy
The governance model improves review discipline for future security and privacy-sensitive changes.

## Graduation Criteria
Architecture governance is used by future AEPs, releases, and conformance reviews.
