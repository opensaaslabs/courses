# Release Policy

## Release Types

- Experimental: early work, no compatibility promise.
- Alpha: usable for feedback, breaking changes expected.
- Beta: nearing stability, breaking changes require justification.
- Stable: compatibility guarantees apply.
- Canonical: recommended baseline for ecosystem adoption.

## Requirements

Every release must include:

- Specification version
- Schema versions
- Contract versions
- Conformance expectations
- Known limitations
- Migration notes
- Security notes
